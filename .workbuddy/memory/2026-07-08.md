## 环境配置：Node.js/npm 加入用户 PATH
- 用户在 PowerShell 运行 `npm run dev` 报 "无法将 npm 项识别为 cmdlet"。
- 原因：WorkBuddy 托管的 Node.js（`C:\Users\user\.workbuddy\binaries\node\versions\22.22.2`，含 node.exe / npm.cmd）未加入用户级 PATH，仅在 WorkBuddy bash 会话可用。
- 解决：用 `[Environment]::SetEnvironmentVariable("Path", ..., "User")` 将该目录追加到用户 PATH 末尾（用户级，无需管理员权限）。
- 注意：环境变量修改对已打开的 PowerShell 窗口不生效，需重开窗口。
